## [Slide 2] Problem — 0:10〜0:35

**"Here's the problem I kept running into."**

"You have multiple AI agents. You give them all the same task.
They all return something. But — which one actually did the best work?
And more importantly — how do you make sure the best one gets used next time?"

"Today, there's no standard answer to that.
Reward is guesswork. Selection is manual. And accountability? Zero."

---

## [Slide 3] Solution — 0:35〜1:05

**"Rutsubo fixes that."**

"Rutsubo is a protocol that creates a competitive economy for AI agents.

You submit a task. Rutsubo's coordinator breaks it into subtasks and dispatches them to multiple agents.
Each result is evaluated by Claude — as an impartial judge — using an asymmetric loss function:
missing critical problems is penalized much harder than being overly cautious.

The scores drive automatic Solana reward distribution.
And every agent's trust score updates after each task — strong agents rise, weak ones fade."

## [Slide 4] Differentiation — 1:05〜1:25

**"Why is this different?"**

"LangGraph, CrewAI — great tools. But they stop at execution.
There's no evaluation loop. No reward signal. No natural selection.

Rutsubo connects evaluation → reward → hiring logic → on-chain record, end to end.
The causal chain endpoint tells you exactly which layer caused a failure —
was it the task definition? The coordinator? The worker?

That transparency is what makes agents accountable."

---

## [Slide 5] Why Me — 1:25〜1:45

**"Why am I the one to build this?"**

"I've been thinking about how to make AI systems that improve themselves —
not just in weights, but in how they're selected and rewarded.

The economic incentive layer is what's missing from most agent frameworks.
Rutsubo is my answer to that: a minimal, composable protocol
that any agent can plug into and immediately start earning based on merit."
